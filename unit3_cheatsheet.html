05/10/14 - JavaScript Objects

1. Objects and properties

An object in JavaScript is any unordered collection of key-value pairs. A property of an object is a variable attached to the object which describes the characteristics of the object. A property name can be any valid JavaScript string or anything that can be converted to a string.

You can access object properties with dot-notation if the property is a valid identifier or bracket notation. Use bracket notation when property names are to be dynamically determined.

Use bracket notation with for..in loops to iterate over enumerable properties. You'll find an example implementation at GitHub.

var myCar = new Object();
myCar.make = "BMW";
myCar["make"] = "BMW";

//or using a string value stored in a variable:
var propertyName = "make";
myCar[propertyName] = "BMW";

2. Enumerating all properties of an object

Here are three ways to list/traverse object properties:

  • for...in loops
    This method traverses all enumerable properties of an object and its prototype chain

  • Object.keys(obj)
    This method returns an array with all the own (not in the prototype chain) enumerable properties' names ("keys") of an object obj

  • Object.getOwnPropertyNames(obj)
    This method returns an array containing all own properties' names (enumerable or not) of an object

3. Create new objects

3.1 Use object initializer (literal notation)

The var obj is the new object; its properties can be an identifier (property_1), a number, or a string:

var obj = { property_1: value_1,
 2: value_2,
 "property n": value_n };

You can also use literal notation to create arrays:

var coffees = ["French Roast", "Colombian", "Kona"]; //array with 3 elements and lenght of 3
var fish = ["Lion", , "Angel"]; //array with 3 elements; fish[1] is undefined

3.2 Use constructor function

You can also create an object using:

  • Constructur function
function Car(make, year) {
 this.make = make; //using this to assign values to the object's properties
 this.year = year;
};
  • Using new
var myCar = new Car("BMW", 1993); //instantiate a new Car object (calling the function Car(make, year))

3.3 Use Object.create method

The Object.create() method creates a new object with a specified prototype object and properties.

var person = Object.create(null);  // the parameter 'null' is the object's prototype (undefined in this example)

4. Object.create() and Prototypes

JavaScript objects are pairs of keys and values. But they also have an additional attribute: a pointer to another object which is called prototype. Prototypes can be used to inherit functionality. We can create a new object with a set of properties, but with a PARTICULAR prototype. You can look up an object's prototype using the method Object.getPrototypeOf(obj):

var woman = Object.create(null);
woman["sex"] = "female";  // setting new property sex eql to "female"

var anna = Object.create(woman);
anna["firstName'] = "Anna";  // setting new property firstName eql to "Anna"
anna["lastName"] = "Morris";

anna.firstName  // "Anna"
anna.sex  // "female"

Object.getPrototypeOf(anna)  // returns the woman object, i.e. { sex: 'female' }


var hanna = Object.create(woman);
hanna["firstName"] = "Hanna";  // setting new property firstName eql to "Anna"
hanna["lastName"] = "Edward";

hanna.firstName  // "Hanna"
hanna.sex  // "female"

Object.getPrototypeOf(hanna)  // returns the woman object, i.e. { sex: 'female' }

In the example above, we created two new objects (anna and hanna) which have individual properties (firstName, lastName) but they share the SAME prototype (woman) as both objects are created with this prototype:

var woman = Object.create(null);
woman.sex = "female";  // setting new property sex eql to "female"

var anna = Object.create(woman);
Object.getPrototypeOf(anna)  // returns the woman object, i.e. { sex: 'female' }

var hanna = Object.create(woman);
Object.getPrototypeOf(hanna)  // returns the woman object, i.e. { sex: 'female' }
